# Loomen

**A fully local personal knowledge management (PKM) app** — it keeps your notes in plain `.md`
files and ships with an embedded daily planner, calendar and Pomodoro timer.

No server, no account, no subscription. Your notes live on your disk, in your folder, in an open format.

- **Platforms:** macOS · Windows · Linux · iOS · Android (Tauri 2.0)
- **License:** [PolyForm Noncommercial 1.0.0](LICENSE) — free for personal use, commercial use prohibited
- **Status:** Active development (`v0.1.0`) — no published release yet
- **Domain:** loomen.org

## Why Loomen?

Most note apps either keep your notes on their own servers, lock them into a proprietary format,
or leave planning to a separate application. Loomen rejects all three:

- **You own your data.** Notes are plain Markdown + YAML frontmatter. Delete the app and your notes remain.
- **No internet required.** None of the core features touch the network.
- **Planning lives inside note-taking.** Tasks, calendar and Pomodoro are not a separate app — they sit next to your notes.

## Features

### Notes and editor
- Markdown editor built on CodeMirror 6, with **live preview**
- **Bidirectional linking** — `[[wiki-link]]` and a backlinks panel
- Table editing widget, formatting toolbar, context menu
- Plain-text editor mode (for those who don't want Markdown syntax)
- Full-text search
- **Daily note** — created automatically from a date template
- **Trash** — deleted notes are kept for 30 days and can be restored

### Planner, calendar and Pomodoro
- Daily plan view, task board and timeline
- Calendar card, mini agenda and stat cards
- Task detail: subtasks, recurrence, notes
- Quick add for tasks
- Built-in **Pomodoro** timer

### Visualization
- **Graph view** — a link graph between notes (d3-force)
- **Drawing** — Excalidraw integration
- Reports screen

### Audio
- In-app **voice recording** (VoiceRecorder), WAV and FLAC encoding
- Audio player embedded in notes

### Synchronization (optional)
We have **no** sync server of our own. Because the vault is plain `.md`, it syncs with
iCloud / Syncthing / Git. In addition, from inside the app:
- **GitHub sync** — OAuth Device Flow *(git-based sync is desktop-only)*
- **Google Calendar** — OAuth 2.0 loopback + PKCE

### Multilingual
Turkish · English · **Arabic (first-class RTL)**. The architecture scales to 20 languages without code changes.

## Development

**Requirements:** Node.js 20+, Rust (stable), and the
[Tauri prerequisites](https://tauri.app/start/prerequisites/) for your platform.

```bash
git clone https://github.com/ofarukerol/Loomen.git
cd Loomen
npm install
cp .env.example .env      # for integrations (optional, see below)
npm run tauri dev
```

Other commands:

```bash
npm run build             # tsc typecheck + Vite build
npm run tauri build       # distributable desktop bundle
```

### Configuration

The GitHub and Google Calendar integrations require your own OAuth client. Step-by-step setup
is documented in `.env.example` — copy it to `.env` and fill it in. **These integrations are
optional; the app works fully without them.**

### Things worth knowing

- If you change `src-tauri/capabilities/` or anything on the Rust side, **restart** the app —
  those are compiled into the binary and are not picked up by HMR.
- Use **pointer events** for drag and drop; Tauri's WKWebView does not reliably support
  native HTML5 DnD.
- `src-tauri/gen/` is not under version control; it is generated by `tauri ios init` / `tauri android init`.

## Architecture

**Frontend:** React 19 · TypeScript · Zustand · CodeMirror 6 · i18next · Excalidraw · d3-force
**Backend (local):** Rust / Tauri 2.0 — file system, OAuth (PKCE), GitHub Git Data API

There is **no** server component and no database. The single source of truth is the `.md` files
on disk; everything in memory is a cache.

> ⚠️ Every change that touches note content must follow the rules in
> [`NOTE_SAFETY_RULES.md`](NOTE_SAFETY_RULES.md). Those rules were written after a real data-loss incident.

### Planning documents
