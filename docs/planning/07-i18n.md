## 1. Temel İlkeler (değişmez kurallar)

1. **Kaynak dil = Türkçe.** Tüm metinler önce Türkçe yazılır; diğer diller bundan **türetilir**.
2. **Hardcoded string YASAK.** UI'da görünen her metin bir **çeviri anahtarı** (key) üzerinden gelir.
   Kod review'da hardcoded metin reddedilir (bkz `§8` lint kuralı).
3. **Anahtar = sözleşme, metin = veri.** Kod anahtara bağlıdır (`planner.focus.todo`), metne değil.
   Böylece çeviri değişse de kod değişmez; yeni dil eklemek **sadece veri** eklemektir.
4. **Otomatik çeviri + insan düzeltmesi (override).** EN/AR ilk hâli makine çevirisiyle **build-time**
   üretilir; sonradan elle düzeltilen anahtarlar **"locked"** işaretlenir, bir daha üzerine yazılmaz.
5. **Runtime'da ağ yok.** Çeviri üretimi **geliştirme/build aşamasında** olur; uygulama çalışırken
   hazır JSON dosyalarını okur. Bu, "tamamen lokal" (NFR-8) ilkesini bozmaz.
6. **RTL ilk günden.** Arapça olduğu için yön (dir) ve simetri en baştan çözülür; sonradan eklenmez.
7. **20 dile sıfır-sürtünme.** Yeni dil eklemek = bir `locale` kodu + bir çeviri klasörü. Kodda
   hiçbir değişiklik gerekmez.

## 2. Mimari Genel Bakış

```mermaid
flowchart TD
    SRC[locales/tr/*.json\n(KAYNAK — elle yazılır)] --> GEN[Çeviri Üretici\n(build-time script + çeviri API)]
    GEN --> EN[locales/en/*.json]
    GEN --> AR[locales/ar/*.json]
    GEN -.gelecekte.-> XX[locales/<20 dil>/*.json]
    LOCK[(translation-memory\n+ locked keys)] <--> GEN
    EN & AR & SRC --> RT[Runtime: i18next\n(react-i18next + ICU)]
    RT --> UI[React UI\n(useTranslation)]
    RT --> DIRX[dir/RTL yöneticisi]
    RT --> FMT[Intl / date-fns\n(tarih, sayı, çoğul)]
```

- **Kaynak (`locales/tr`)** elle yazılır → tek doğruluk kaynağı.
- **Üretici script** TR'yi tarar, eksik/değişmiş anahtarları EN/AR (ve gelecekte 17 dil) için çevirir.
- **Translation memory** zaten çevrilmiş/elle düzeltilmiş anahtarları hatırlar → mükerrer çeviri yok,
  elle düzeltmeler korunur.
- **Runtime** sadece hazır JSON okur; i18next anahtarı metne çevirir, ICU çoğul/cinsiyet, Intl/date-fns
  biçimlendirme, dir yöneticisi RTL'yi halleder.

## 3. Dosya Yapısı & Namespace'ler

Özellik bazlı **namespace** (özellik başına ayrı dosya) → büyük projede yönetilebilirlik + lazy-load.

```
src/i18n/
├── index.ts                  # i18next başlatma, dil algılama, fallback zinciri
├── config.ts                 # SUPPORTED_LOCALES, DEFAULT, RTL_LOCALES
├── dir.ts                    # RTL/LTR yön yöneticisi
└── locales/
    ├── tr/                   # KAYNAK (elle)
    │   ├── common.json       # genel: kaydet, iptal, sil, evet, hayır...
    │   ├── editor.json
    │   ├── explorer.json
    │   ├── planner.json      # Focus On Today, Todo, Overdue, Unplanned, Inbox...
    │   ├── calendar.json
    │   ├── pomodoro.json
    │   ├── search.json
    │   ├── graph.json
    │   └── settings.json
    ├── en/                   # türetilmiş (otomatik + override)
    │   └── ...aynı dosyalar
    └── ar/                   # türetilmiş + RTL
        └── ...aynı dosyalar
```

### Anahtar isimlendirme sözleşmesi
- Biçim: `namespace:alan.altAlan` → `planner:focus.overdue`, `pomodoro:timer.start`.
- **Sadece anahtar, asla cümle, anahtar olarak kullanılmaz** (örn. `t("Kaydet")` ❌ → `t("common:save")` ✅).
- İç içe (nested) yapı; düz "anahtar = uzun cümle" değil.

### Örnek `tr/planner.json`
```json
{
  "focus": {
    "title": "Bugüne Odaklan",
    "todo": "Yapılacak",
    "overdue": "Geciken",
    "unplanned": "Planlanmamış"
  },
  "inbox": {
    "placeholder": "Görevini buraya yaz...",
    "added": "{{count}} görev eklendi"
  },
  "relative": {
    "today": "bugün",
    "tomorrow": "yarın",
    "yesterday": "dün",
    "inDays": "{{count}} gün sonra",
    "daysAgo": "{{count}} gün önce"
  }
}
```

## 4. Çoğul, Değişken, Cinsiyet (ICU MessageFormat)

20 dilde doğru gramer için düz string birleştirme **yetmez** (Arapça'da 6 çoğul biçimi vardır!).
Çözüm: **ICU MessageFormat** (`i18next-icu`).

```json
"taskCount": "{count, plural, =0 {görev yok} one {# görev} other {# görev}}"
```
- Arapça: `zero / one / two / few / many / other` — ICU bunu locale verisinden otomatik seçer.
- Tarih/sayı: **asla elle formatlanmaz** → `Intl.DateTimeFormat` / `Intl.NumberFormat` / `date-fns` locale.
- **String concatenation ile cümle kurmak YASAK** (kelime sırası dile göre değişir). Tüm değişkenler
  ICU placeholder ile (`{name}`, `{count}`) gömülür.

## 5. RTL / Sağdan-Sola (Arapça — birinci sınıf)

| Konu | Kural |
|------|-------|
| Yön | `<html dir="rtl">` aktif locale RTL ise; `dir.ts` dili değişince otomatik ayarlar |
| CSS | **Logical properties zorunlu:** `margin-inline-start` (≠ `margin-left`), `padding-inline`, `inset-inline`. Fiziksel `left/right` **yasak** |
| İkonlar | Yön bildiren ikonlar (◀ ▶, geri/ileri, ağaç oku) RTL'de **aynalanır** |
| Layout | Sol panel RTL'de sağa geçer; takvim/timeline yönü ters akar — logical props ile otomatik |
| Karışık metin | Türkçe içine Arapça (veya tersi) gömülünce **bidi** izolasyonu (`&#8235;`/CSS `unicode-bidi: isolate`) |
| Sayılar | Arapça-Hint rakamları mı Batı rakamı mı → locale tercihine bırak (Intl) |
| Test | Her ekran hem LTR hem RTL'de görsel doğrulanır (`05` cross-platform genişletilir) |

> **Tailwind kullanılırsa:** RTL eklentisi / logical utility'ler (`ms-*`, `me-*`, `ps-*`, `pe-*`)
> tercih edilir; `ml-*`/`mr-*` gibi fiziksel sınıflardan kaçınılır.

## 6. Otomatik Çeviri Pipeline'ı (TR → diğer diller)

Kullanıcının "diğer diller otomatik oluşacak" talebinin teknik karşılığı.

### Akış
1. Geliştirici yalnızca **`locales/tr/*.json`**'u günceller (yeni anahtar / değişen metin).
2. `npm run i18n:sync` çalışır:
   - TR'deki her anahtarı diğer dillerle karşılaştırır.
   - **Eksik** anahtarları çeviri API'siyle (build-time, ör. DeepL/Google/LLM) çevirir.
   - **TR'de değişmiş** anahtarları yeniden çevirir — **ama** hedef dilde `"_locked": true` ise dokunmaz.
   - **Translation memory** (`.i18n-cache.json`) ile aynı metni iki kez çevirmez (maliyet + tutarlılık).
3. Sonuç JSON'lar repoya commit'lenir → uygulama runtime'da bunları okur (ağsız).

### Anahtar durumu (her hedef dil anahtarı için meta)
```json
"focus": { "todo": "To Do", "todo_meta": { "state": "auto", "srcHash": "a1b2", "locked": false } }
```
- `state`: `auto` (makine) | `human` (elle düzeltilmiş) | `stale` (kaynak değişti, yeniden çevrilecek).
- `srcHash`: kaynak TR metnin hash'i → kaynak değişince `stale` olur.
- `locked`: elle düzeltilen çeviriyi korur (otomatik üzerine yazmaz).

> Bu mekanizma sayesinde **3→20 dil** geçişinde tek yapılacak: `config.ts`'e locale kodu eklemek +
> `i18n:sync` çalıştırmak. Geri kalanı otomatik; elle düzeltmeler korunur.

## 7. Runtime Davranışı

- **Dil seçimi:** ayarlardan manuel; ilk açılışta OS diline göre öneri, yoksa **TR varsayılan**.
- **Fallback zinciri:** `seçili dil → TR (kaynak)`. (EN'e değil; kaynak TR'ye düşülür ki anlam kaybolmasın.)
- **Lazy-load:** namespace'ler ihtiyaç anında yüklenir (graph açılmadan `graph.json` yüklenmez) → hız.
- **Eksik anahtar:** dev'de görünür uyarı (`⟦missing: planner:focus.todo⟧`), prod'da TR fallback.
- **Anında değişim:** dil değişince uygulama yeniden başlamadan tüm metinler + `dir` güncellenir.

## 8. Kalite & Sürdürülebilirlik Kuralları

- [ ] **Lint:** JSX içinde çıplak metin (hardcoded string) için ESLint kuralı (`i18next/no-literal-string`).

- [ ] **Tarih/sayı:** her zaman Intl/date-fns locale; elle biçim **yok**.
- [ ] **RTL testi:** her yeni ekran LTR + RTL görsel kontrolünden geçer.
